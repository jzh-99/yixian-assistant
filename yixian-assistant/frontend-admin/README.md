# 翼线助手后台管理系统 V1.0

根据《翼线助手后台管理系统产品需求文档 V1.0》搭建的 PC Web 后台。项目默认以本地演示模式运行，完整覆盖申请审批、人员技能、智能派单、任务监控、统计分析、管理智脑、字典和操作日志。

## 快速运行

```bash
npm install
npm run dev
```

浏览器访问 `http://localhost:4173`。

生产构建：

```bash
npm run build
npm run preview
```

## 演示账号

| 账号 | 密码 | 角色 | 数据范围 |
| --- | --- | --- | --- |
| `admin` | `123456` | 超级管理员 | 全省组织 |
| `dept` | `123456` | 部门管理员 | 本部门及下级 |
| `observer` | `123456` | 只读观察员 | 授权组织，只读 |

系统设置页提供“复位演示数据”，可以恢复审批、派单、人员和日志的初始状态。

## 已实现模块

- 运营工作台：待审批、待派单、进行中、完成率、任务趋势、团队负载和异常任务。
- 审批中心：三类 APP 申请、筛选、详情、通过、驳回、超时提示和审批轨迹。
- 人员与技能：人员状态、技能标签、并发上限、当前负载、停用前任务校验。
- AI②任务派单：硬约束过滤、技能与负载评分、推荐解释、人工确认和派单记录。
- 三色监控：已超时、临近到期、正常、时间缺失，支持任务详情和 APP 状态时间线。
- 装维效率：完成率、处理时长、超时率、个人排行、技能负载和明细穿透。
- 拜访与商机：拜访趋势、转化漏斗、经理排行、高意向客户和 APP 拜访记录。
- AI④管理智脑：受控模板问答，返回指标、排行、表格、口径、时间和数据范围。
- 系统设置：字典维护、全局阈值展示、不可修改的操作日志。

## 数据模式

默认配置：

```env
VITE_API_MODE=mock
```

演示模式使用浏览器 `localStorage` 保存业务变更，页面刷新后审批和派单结果仍然存在。

连接现有 APP Spring Boot 服务：

```env
VITE_API_MODE=remote
VITE_API_BASE_URL=/api/v1
VITE_API_TARGET=http://localhost:8080
VITE_API_TOKEN=<联调令牌>
```

远程模式会同步 `/applications`、`/tasks`、`/visits` 和 `/opportunities`，并与后台演示数据按 ID 合并。详细契约见 [APP数据对接说明.md](./APP数据对接说明.md)。

## 目录

```text
src/
  components/   通用图表、状态、指标卡和管理智脑
  layouts/      后台主框架与权限菜单
  router/       路由与登录守卫
  services/     会话、演示数据、业务动作、APP 数据同步
  views/        各业务页面
```

## 1.0 边界

- 组织树、人员主数据和客户负责关系采用固定演示数据。
- 管理智脑采用受控模板，不调用任意 SQL。
- 前端已预留后台审批、派单和改派写接口；现有 APP 后端尚未提供这些管理员接口时，演示环境继续使用本地闭环。
- 生产环境必须由后端强制校验菜单权限、操作权限和组织数据范围。
