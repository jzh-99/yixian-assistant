# AGENTS.md — 翼线助手 AI 模块说明

> 本项目为 AI Coding 课程项目，使用 DeepSeek API 实现四个 AI 模块。
> 所有 AI 调用均走后端，前端不直接持有 API Key。

---

## AI① 智能填单（2号：程言欣）

**作用**：将用户自然语言输入转换为结构化申请表单字段，并主动检测异常场景。

**接口**：`POST /api/v1/ai/extract`

**Prompt 策略**：
- System：定义字段 Schema、申请类型枚举、异常检测规则（周末/数量超限/时间过紧）
- User：用户输入的自然语言 + 申请类型
- 输出：JSON，含 fields、missingFields、warnings、confidence

**降级策略**：AI 不可用时返回空字段，前端展示普通手工填单入口。

---

## AI② 智能派单理由（4号：戈世奇）

**作用**：算法选出最优执行人后，调用 DeepSeek 生成一句自然语言推荐理由，展示在派单结果旁边。

**接口**：`POST /api/v1/ai/dispatch-reason`

**Prompt 策略**：
- 输入：申请描述 + 被选人员数据（技能、当前任务数、30天完成率）
- 输出：一句推荐理由，20-50 字
- 算法负责"选谁"，LLM 负责"为什么"，两者分工明确

**注意**：LLM 不决定派单结果，只生成解释文案。

## AI③ 智能助手（3号：娄登科）

**作用**：为一线员工提供业务规则解释、流程指引和操作建议。

**接口**：`POST /api/v1/ai/chat`

**Prompt 策略**：
- System：注入 20 条业务 FAQ（审批流程、物料标准、常见故障等）+ 角色说明
- 优先引用 FAQ 内容回答，不确定时提示"建议联系主管确认"
- 返回 suggestedAction 字段，支持跳转到对应业务页面

**FAQ 维护**：内容由业务同学审核，版本和更新时间记录在 system prompt 头部。

## AI④ 管理智脑（6号：蒋子涵）

**作用**：让管理者用自然语言查询运营指标，返回指标卡、表格或排行。

**接口**：`POST /api/v1/ai/query`

**设计原则**：
- 禁止 LLM 直接写 SQL 或访问数据库
- 意图识别后匹配白名单模板，参数化查询
- 权限范围由后端从登录态注入，模型无法扩大数据范围

**支持的 5 个意图**：

| intentId | 说明 | 示例问法 |
|----------|------|---------|
| COMPLETION_RATE | 装维任务完成率 | 本周完成率是多少？ |
| OVERTIME_RANK | 超时任务排行 | 哪个团队超时最多？ |
| TASK_COUNT | 任务量统计 | 张师傅本周接了几单？ |
| VISIT_STATS | 拜访次数统计 | 王经理本月拜访了几次？ |
| HIGH_INTENT_OPPORTUNITIES | 高意向商机 | 现在有哪些高意向客户？ |

**降级**：意图未命中时返回标准问法提示，不返回错误信息。

**安全**：
- 所有参数化，无字符串拼接 SQL
- 审计日志记录每次查询（用户、意图、耗时、结果行数）
- 敏感字段（手机号）默认脱敏

## AI Coding 开发方式

- 工具：Cursor / Windsurf
- 策略：先写 Prompt，在 DeepSeek 网页端测试效果，确认后再写接口代码
- 所有 AI 接口统一经过后端，API Key 通过环境变量注入，不出现在代码和 git 仓库
