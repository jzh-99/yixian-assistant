# 翼线助手 · 2号审批模块开发说明


---

## 一、模块职责（2号）

负责 **申请提交、审批流转、AI① 智能填单**，是 APP 装维/客户经理与 PC 后台审批中心之间的核心链路。

| 功能 | 接口 | 说明 |
|------|------|------|
| 提交申请 | `POST /api/v1/applications` | APP 提交领料/拜访/支撑申请 |
| 我的申请 | `GET /api/v1/applications` | 分页列表，一线员工只看自己的 |
| 待审批列表 | `GET /api/v1/approvals/pending` | 管理员/调度员查看待办 |
| 审批通过 | `POST /api/v1/approvals/{id}/approve` | 写入审批记录，必要时创建任务 |
| 审批驳回 | `POST /api/v1/approvals/{id}/reject` | 驳回原因 5–500 字 |
| 审批轨迹 | `GET /api/v1/approvals/{id}/records` | 查看历史审批记录 |
| AI 智能填单 | `POST /api/v1/ai/extract` | 自然语言 → 结构化字段 |
| 撤回申请 | `POST /api/v1/applications/{id}/cancel` | 仅 `PENDING` 状态可撤回 |

PC 后台前端兼容路径（与上面 approve/reject 逻辑相同）：

- `POST /api/v1/admin/applications/{id}/approve`
- `POST /api/v1/admin/applications/{id}/reject`

## 三、业务规则

### 3.1 申请类型

| type | 含义 | 审批通过后 |
|------|------|-----------|
| `MATERIAL` | 领料申请 | 创建 `UNASSIGNED` 任务（供 4 号派单） |
| `SUPPORT` | 支撑申请 | 创建 `UNASSIGNED` 任务 |
| `VISIT` | 外出拜访 | 不创建任务，直接生效 |

### 3.2 申请状态流转

```
PENDING → APPROVED   （审批通过）
PENDING → REJECTED   （驳回，需填写原因）
PENDING → CANCELLED  （申请人撤回）
```

### 3.3 权限控制

| 角色 | 可执行操作 |
|------|-----------|
| `MAINTAINER` / `ACCOUNT_MANAGER` | 提交申请、查看自己的申请、撤回 |
| `SUPER_ADMIN` / `DISPATCHER` | 查看全部、审批通过/驳回 |
| `DEPT_ADMIN` | 仅本部门申请的审批 |

### 3.4 写操作幂等

所有 POST 写接口请求体需携带 `idempotencyKey`，24 小时内重复提交会返回「请勿重复提交」。

## 十一、接口路径协调（已解决）

| 来源 | 路径 |
|------|------|
| 2号 TODO / 架构契约 | `POST /api/v1/approvals/{id}/approve` |
| PC 前端 `remoteApi.js` | `POST /api/v1/admin/applications/{id}/approve` |
| APP 数据对接说明 | `POST /api/v1/admin/applications/{id}/approve` |

**协调结论：后端已做路径别名，两条路径指向同一方法，无需改前端。**

```java
// ApplicationController.java
@PostMapping({"/approvals/{id}/approve", "/admin/applications/{id}/approve"})
@PostMapping({"/approvals/{id}/reject",  "/admin/applications/{id}/reject"})
```

| 建议 | 说明 |
|------|------|
| PC 前端 | 保持 `remoteApi.js` 现有 `/admin/applications/...` 不变 |
| APP / 接口测试 | 可用 `/approvals/...` 或 `/admin/applications/...` 均可 |
| 驳回接口 | 同上，两条路径等价 |

> 注意：`ApprovalsView.vue` 目前仍调用 mockStore，远程模式需在页面里改用 `adminWriteApi.approve/reject` 才能真正联调。
