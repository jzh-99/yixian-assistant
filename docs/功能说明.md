# 装维功能说明

## 1. 装维工作台

装维人员登录后默认进入工作台（`OpsHomeScreen`），主要能力：

- **任务分类统计**：待接单、已接单、进行中、已完成四个 Tab，显示各状态任务数量
- **超时提醒**：超过期望完成时间的任务卡片红色高亮，显示超时文案
- **快速领料**：一键跳转 AI 智能填单页
- **任务卡片操作**：
  - 待接单 → 按钮「接单」
  - 已接单 → 按钮「开始工作」
  - 进行中 → 按钮「反馈完工」
  - 已完成 → 按钮「查看详情」

## 2. AI 领料填单

流程分为两步：

### 2.1 智能填单（SmartFormScreen）

1. 装维人员用自然语言描述领料需求（支持语音模拟输入）
2. 调用 `POST /api/v1/ai/extract`，传入 `applicationType: MATERIAL`
3. AI 返回结构化字段：事由、地址、期望时间、紧急程度、物料清单
4. 低置信度字段标记为「待确认」

### 2.2 确认领料申请（MaterialPreviewScreen）

1. 展示 AI 解析结果，用户可增删改物料行
2. 必填校验：事由、地址、期望时间、物料名称与数量
3. 提交 `POST /api/v1/applications`，类型 `MATERIAL`
4. 成功后跳转申请详情

**示例输入**：「明天去鼓楼区修宽带，领一台光猫和一根光纤。」

## 3. 接单

- **触发**：工作台或任务详情页点击「接单」
- **确认弹窗**：显示工单编号和任务类型
- **API**：`POST /api/v1/tasks/{id}/accept`，请求体 `{ version, idempotencyKey }`
- **状态变更**：`PENDING_ACCEPTANCE` → `ACCEPTED`
- **乐观锁**：后端校验 `version`，冲突返回 409

## 4. 开始工作

- **触发**：已接单任务点击「开始工作」
- **API**：`POST /api/v1/tasks/{id}/start`
- **状态变更**：`ACCEPTED` → `IN_PROGRESS`

## 5. 完工反馈

- **触发**：进行中任务点击「反馈完工」，进入 `TaskFeedbackScreen`
- **必填项**：
  - 实际用料（文本）
  - 完工备注（文本）
  - 客户手写签名（Canvas 签名板）
- **选填项**：现场图片，最多 3 张
- **提交流程**：
  1. 上传图片 → `POST /api/v1/files/upload`（bizType: TASK_FEEDBACK）
  2. 上传签名 → `POST /api/v1/files/upload`（bizType: TASK_SIGNATURE）
  3. 提交完工 → `POST /api/v1/tasks/{id}/complete`
- **状态变更**：`IN_PROGRESS` → `COMPLETED`
- **数据持久化**：写入 `task_feedback` 表

## 6. 关键数据结构

### 任务 TaskEntity

```java
taskNo, type, title, assigneeId, status,
customerName, contactPhone, address, materialsJson,
expectedFinishTime, actualFinishTime, version
```

### 完工反馈 TaskFeedback

```java
taskId, operatorId, actualMaterials, resultDesc,
imageUrlsJson, signatureUrl
```

### 领料申请 extra 字段

```json
{
  "location": "南京市鼓楼区",
  "expectedTime": "2026-06-11 09:00:00",
  "urgency": "NORMAL",
  "materials": [{ "name": "光猫", "quantity": 1, "unit": "台" }]
}
```
