## 二、接口统一规范

### 2.1 统一响应格式（所有接口必须返回此格式）

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "uuid-xxxx"
}
```

- `code = 0` 表示成功，其余为错误
- HTTP 状态码**统一返回 200**，业务错误通过 `code` 区分
- `requestId` 每次请求唯一，用于日志追踪

### 2.2 错误码

| code | 含义 |
|------|------|
| 0 | 成功 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 并发冲突（version 不匹配） |
| 1001 | 参数校验失败 |
| 1002 | 业务状态异常（如重复审批） |
| 2001 | AI 服务不可用，已降级 |
| 500 | 服务器内部错误 |

### 2.3 分页

请求参数：`pageNo`（从 1 开始）、`pageSize`（默认 20，最大 100）

```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "pageNo": 1,
    "pageSize": 20
  }
}
```

### 2.4 命名规范（全员统一，不得混用）

| 位置 | 规范 | 示例 |
|------|------|------|
| 接口路径 | 小写 + 短横线 | `/api/v1/dispatch-records` |
| JSON 字段 | 小驼峰 | `userId`、`createTime` |
| 数据库字段 | 下划线 | `user_id`、`create_time` |
| 时间格式 | `yyyy-MM-dd HH:mm:ss` | `2026-06-11 14:30:00` |
| 金额单位 | 整数，单位分 | `12000` 表示 120.00 元 |

### 2.5 认证

所有接口（除 `/auth/login`）需携带 Header：

```
Authorization: Bearer <token>
```

后端统一拦截器校验，业务代码中不重复写鉴权逻辑。

### 2.6 幂等

所有写操作（提交、审批、派单、完工）客户端生成 `idempotencyKey` 随请求提交，后端防重复处理。

## 七、AI 模块规范

### 7.1 AI① 智能填单

**接口**：`POST /api/v1/ai/extract`

请求：
```json
{
  "input": "明天上午去江宁修光缆，带50米光缆和熔接机",
  "applicationType": "MATERIAL"
}
```

响应 data：
```json
{
  "fields": {
    "title": "江宁区光缆维修",
    "content": "前往江宁区进行光缆熔接作业",
    "expectedTime": "2026-06-12 09:00:00",
    "location": "江宁区",
    "materials": [
      {"name": "光缆", "quantity": 50, "unit": "米"},
      {"name": "熔接机", "quantity": 1, "unit": "台"}
    ]
  },
  "missingFields": ["contactName"],
  "warnings": [
    {"field": "expectedTime", "message": "申请日期为周六，是否确认？"}
  ],
  "confidence": 0.92
}
```

- `warnings` 为空数组时前端不展示提示卡
- `confidence < 0.6` 时前端高亮所有字段要求用户逐一确认

### 7.2 AI② 派单推理理由

**接口**：`POST /api/v1/ai/dispatch-reason`

请求：
```json
{
  "taskId": 123,
```

响应 data：
```json
{
  "reason": "推荐张师傅：具备光缆熔接技能，当前无进行中任务，近30天完成率98%，综合评分最优。"

```

### 7.3 AI③ 智能助手

**接口**：`POST /api/v1/ai/chat`

请求：
```json
{
  "message": "领料申请需要提前几天提交？",
  "sessionId": "uuid-xxxx",
  "history": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

响应 data：
```json
{
  "answer": "领料申请需提前1个工作日提交，紧急情况可联系组长走加急通道。",
  "source": "领料申请管理规范 v2.1",

```

- `confidence < 0.5` 时 answer 末尾追加"建议联系主管确认"
- `suggestedAction` 为 null 时前端不展示跳转按钮

### 7.4 AI④ 管理智脑

**接口**：`POST /api/v1/ai/query`

请求：
```json
{
  "question": "本周江宁组超时任务最多的是哪个人？",
  "sessionId": "uuid-xxxx"
}
```

响应 data：
```json
{
  "intentId": "OVERTIME_RANK",
  "params": {
    "timeRange": "THIS_WEEK",
    "deptName": "江宁组",
    "topN": 1
  },
  "result": {
    "type": "TABLE",
    "columns": ["姓名", "超时任务数", "部门"],
    "rows": [["李工", 3, "江宁装维班组"]]
  },
  "dataScope": "江宁装维班组",
  "timeRange": "2026-06-08 ~ 2026-06-11",
  "updatedAt": "2026-06-11 14:30:00"
}
```

**5个白名单意图：**

| intentId | 说明 | 关键参数 |
|----------|------|---------|
| `COMPLETION_RATE` | 装维任务完成率 | timeRange, deptId |
| `OVERTIME_RANK` | 超时任务排行 | timeRange, deptId, topN |
| `TASK_COUNT` | 任务量统计 | timeRange, userId/deptId |
| `VISIT_STATS` | 拜访次数统计 | timeRange, managerId |
| `HIGH_INTENT_OPPORTUNITIES` | 高意向商机列表 | topN |

意图未命中时返回：
```json
{
  "intentId": null,
  "result": null,
  "suggestion": "暂不支持该查询，您可以试试：本周完成率是多少？哪个团队超时任务最多？"
}
```
